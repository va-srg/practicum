#### [ad‑hoc кейс: гео‑сегментация пользователей для маркетинговой кампании в сфере DM&E (Digital Media & Entertainment)](https://github.com/va-srg/practicum/blob/main/Ad‑hoc%20analysis%3A%20user%20geo‑segmentation%20for%20a%20marketing%20campaign%20in%20DM%26E/script_melodyset.sql)
ad‑hoc case: transaction analysis for e‑commerce platform

**Цель** - проанализировать паттерны потребления музыкального контента по регионам и жанрам, выявить наиболее востребованные категории для оптимизации маркетинговой кампании стримингового сервиса. **Основной задачей** является написание серии ad‑hoc запросов к БД для расчета ключевых метрик вовлеченности (количество прослушиваний, длительность, доля воспроизведения) и сегментации аудитории по географическому и жанровому признакам

**Структура работы**

python
- подготовлен скрипт с искуственно созданными данными (модуль random) для имитации БД неизвестного музыкального стримингового сервиса

sql
- через дамп в dbeaver загружен датасет искуственной БД melodyset (postgres);
- выполнена серия ad‑hoc запросов к БД для анализа потребления музыкального контента:

    -  получена первичная выборка данных и список уникальных жанров,
    -  определен топ‑5 жанров по количеству прослушиваний,
    -  сформирована статистика по жанрам, выгружены прослушивания жанра «Reggae» в северо‑западном регионе,
    -  проведен сравнительный анализ музыкальных предпочтений регионов,
    -  проанализированы случаи неполного прослушивания композиций,
    -  выполнена категоризация прослушиваний по длительности и классификация композиций по жанровым категориям, а также подготовлены выборки по дням недели и региональному потреблению треков для поддержки решений по маркетинговой кампании
