#### [Анализ продаж маркетплейса](https://github.com/va-srg/practicum/tree/main/Online%20sales%20platform%20dashboard%20development/dashboard%20overview)

Marketplace sales analysis

**Цель проекта** - создать интерактивный дашборд в Apache Superset для мониторинга и анализа ключевых бизнес‑метрик маркетплейса на основе исторических данных. **Основная задача** - реализовать разделы анализа продаж и категорий товаров, активности и удержания пользователей, рейтингов и отзывов, регистраций и активности по времени с визуализацией ключевых метрик и настройкой кросс‑фильтрации с предустановленными значениями периода и гранулярностью по месяцам (выбран последний день месяца включительно), включая специальный чарт анализа покупок / продаж относительно "последнего дня месяца" (от которого идет отсчет в чартах); с помощью css и json применены кастомные настройки внешнего вида визуализаций

**Структура работы**

sql
- выполненны ad-hoc запросы по основным ключевым метрикам, которые использованы в датасетах;
- сформирован объединенный (продажи и активность) и дополнительный датасеты (пользователи) для дальнейшего использования на уровне BI-инструмента на основе таблиц БД (postgresql) - структура которых представлена в er-диаграмме;
- сформирован дополнительный датасет с последовательностью дней на уровне BI-инструмента для реализации сравнения отсчета от выбранного дня по умолчанию на месяц/полгода назад

superset
- созданы виртуальные датасеты на основе подготовленных запросов, также созданы кастомные метрики расчетов на уровне датасета;
- разработана трехстраничная структура дашборда, с дополнительной вкладкой аннотации внутри него (включая обучение новых пользователей);
- настроены фильтры, их значения по умолчанию, продумана и реализована кросс-фильтрация;
- реализованы чарты сравнения отсчета от выбранного дня по умолчанию на месяц/полгода назад с использованием jinja;
- выполнено кастомное преобразование внешнего вида дашборда и его визуализаций
